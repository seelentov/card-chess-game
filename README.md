# Hearthstone: Battlegrounds — Backend (Java Core)

Этот проект представляет собой **backend-реализацию игрового режима Battlegrounds** из популярной карточной игры *Hearthstone*.  
Разработан полностью на **Java Core** без использования Spring или других фреймворков.  
Сервер обрабатывает логику игры, управляет состоянием матчей, реагирует на игровые события и взаимодействует с клиентами через **WebSocket** и **HTTP**.

**Для тестирования игры можно воспользоваться простым клиентом по адресу: http://5.129.232.61:5173. Вся основная логика готова и вы можете сыграть игру со своей компанией**

---

## Основные возможности

- Полная реализация игровой логики Battlegrounds:
  - Фазы покупки, боя, рекрутирования миньонов
  - Механики трибов, аур, боевых характеристик
  - Система уровней таверны и экономики монет
- Поддержка многопользовательских игр (до 8 игроков)
- Обработка пользовательских действий в реальном времени через WebSocket
- HTTP API для метаопераций (создание комнаты, получение статуса игры и т.д.)
- Событийно-ориентированная архитектура внутри игрового движка
- Чистая Java (JDK 17+), без внешних зависимостей

---

## Технологии

- **Язык**: Java 17+
- **Сборка**: Gradle (Kotlin DSL) с использованием Shadow Jar для создания fat JAR
- **Web-фреймворк**: [Javalin](https://javalin.io/) — лёгкий HTTP-сервер для REST API
- **WebSocket**: [Java-WebSocket](https://github.com/TooTallNate/Java-WebSocket) — нативная реализация WebSocket-соединений
- **JSON-обработка**: Jackson (включая Kotlin-модуль для совместимости)
- **Аутентификация**: JWT через [jjwt](https://github.com/jwtk/jjwt)
- **Рефлексия**: [Reflections](https://github.com/ronmamo/reflections) — для сканирования классов во время выполнения (например, при регистрации обработчиков)
- **Логирование**: SLF4J с простым биндингом (`slf4j-simple`)
- **Тестирование**: JUnit 5

Весь проект написан на **чистой Java без Spring**, что обеспечивает минимальные накладные расходы и полный контроль над архитектурой сервера.
